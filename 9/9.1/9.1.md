# 9.1. Revisión de Entradas

## Funcionalidades primarias (Casos de Uso)

Las funcionalidades primarias del servicio consideran las siguientes historias de usuario:

| ID | Módulo | Nombre | Historia de Usuario |
|-------------|----------------------|----------|-----------|
| CU01 | Autenticacón / Seguridad | Creación de usuario | Yo, como usuario cliente, quiero poder crear un usuario para poder interactuar con las funcionalidades de la página. |
| CU02 | Autenticacón / Seguridad | Inicio de sesión | Yo, como usuario cliente, quiero poder iniciar sesion en la pagina para ver las funcionalidades. | 
| CU03 | Itinerario | Ver recomendaciones | Yo, como usuario cliente, quiero poder ver informacion sobre recomendaciones de los servicios ofrecidos para ser agregados al itinerario. |
| CU04 | Reseñas y comentarios | Crear reseña | Yo, como usuario cliente, quiero poder crear una reseña sobre un servicio contratado ofrecido en Turi. |
| CU05 | Reseñas y comentarios | Agregar comentario | Yo, como usuario cliente, quiero poder agregar un comentario a una reseña sobre un servicio contratado ofrecido en Turi. |
| CU06 | Blog | Utilizar el blog | Yo, como usuario cliente, quiero poder utilizar el blog para ver las novedades de la página. |
| CU07 | Servicios | Subscripcion a servicio | Yo, como usuario cliente, quiero poder acceder a un sistema de recomendaciones de servicios. |
| CU08 | Itinerario | Creación de itinerario | Yo, como usuario cliente, quiero poder crear un itenerario con los servicios que me gusten. |

## Escenarios de Atributos de Calidad

| ID | Atributo de Calidad | Escenario | Caso de uso asociado |
|-------------|----------------------|----------|-----------|
| QA01 | Fiabilidad |Si ocurre un fallo durante la creación de usuario, el sistema debe notificar al cliente y reintentar el proceso automáticamente. | CU01 |
| QA02 | Rendimiento | La creación de un itinerario con servicios debe completarse en menos de 3 segundos por solicitud.| CU08 |
| QA03 | Seguridad | Las funciones unicas (creacion de itinerarios, reservas, favoritos, comentarios, etc) para usuario deben ser accesible únicamente para los usuarios autenticados, utilizando un token de sesión válido.| CU01 |
| QA04 | Rendimiento | El sistema de recomendaciones debe generar resultados personalizados en menos de 2 segundos después de recibir la solicitud del usuario.| CU03 |
| QA05 | Rendimiento | El sistema debe cargar las novedades del blog para cualquier usuario en menos de 3 segundos. | CU06 |
| QA06 | Disponibilidad |El módulo de reseñas debe estar disponible el 99.9% del tiempo. | CU04, CU05 |
| QA07 | Rendimiento |Al agregar un comentario a una reseña, la publicación debe completarse en menos de 3 segundos. | CU05 |
| QA08 | Escalabilidad |Durante un incremento del 50% en el uso del sistema, este debe soportar. | CU03, CU06, CU08 |
| QA09 | Disponibilidad |El sistema debe garantizar que las funcionalidades estén disponibles al menos el 99.9% del tiempo. |  Todos  |
| QA10 | Usabilidad |La interfaz para la creación de usuarios debe ser intuitiva, permitiendo que el 95% de los usuarios nuevos completen el registro sin ayuda. |  CU01  |
| QA11 | Disponibilidad |El sistema debe garantizar la disponibilidad de los itinerarios creados al menos el 99.9% del tiempo. |  CU08  |
| QA12 | Mantenibilidad |El sistema será fácil de modificar para implementar cambios y demorará menos de un día.|  Todos  |

## Restricciones

| ID | Nombre | Descripción |
|-------------|----------------------|-----------|
| CON01 | Idioma |La plataforma estará disponible en idioma español. |
| CON02 | Ley N 29733, Protección de Datos Personales | El sistema debe cumplir con las normas de la ley 29733 para proteger la integridad de los datos de los usuarios. |
| CON03 | Accesibilidad | El sistema debe ser compatible con los navegadores web mas usados |
| CON04 | Sincronización de dispositivos | Los itinerarios creados en un dispositivo deben estar sincronizados automáticamente en todos los dispositivos vinculados a la cuenta del usuario. |
| CON05 | Soporte | Los usuarios cuentan con una seccion de preguntas y respuestas para resolver dudas. |
| CON06 | Actualizacion en tiempo real | La información sobre destinos, actividades y recomendaciones debe ser actualizada dinámicamente para garantizar la relevancia de los datos presentados al usuario. |

## Preocupaciones a nivel de Arquitectura

| ID | Nombre | Descripción |
|-------------|----------------------|-----------|
| CRN01 | Estructura inicial | Determinar una arquitectura modular que permita la integración futura de nuevas funcionalidades. |
| CRN02 | Disponibilidad | Garantizar que la plataforma pueda soportar altas cargas de usuarios en temporadas de turismo y asegurar un tiempo de actividad superior al 99.9%. |
| CRN03 | Conocimientos técnicos del equipo | Aprovechar los conocimientos del equipo en tecnologías web como React. |
| CRN04 | Seguridad y autenticación | Implementar un sistema de seguridad que se alinee a las normativas locales e internacionales de las leyes de protección de datos. |
| CRN05 | Experiencia del usuario | Diseñar una interfaz rápida e intuitiva que sea fácil de entender para los usuarios.|
| CRN06 | Almacenamiento y gestión de datos | Utilizar bases de datos escalables y optimizadas para manejar grandes volúmenes de información en tiempo real.|
| CRN07 | Adaptabilidad a cambios | Adoptar un enfoque ágil que permita implementar modificaciones rápidas en la plataforma en respuesta a cambios en las demandas de los usuarios o el mercado.|
